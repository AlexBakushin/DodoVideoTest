# Прототип системы детекции уборки столиков по видео

Этот проект демонстрирует упрощённый, но рабочий пайплайн анализа видео для отслеживания состояния одного выбранного столика:

- стол пустой;
- стол занят;
- подход к столу после периода пустоты.

В качестве базового решения используется **YOLOv8n** для детекции людей и ручной выбор **одной зоны столика** на видео.  
Результатом работы является видео с визуализацией состояния столика, CSV-файл с событиями и краткая статистика по задержке между уходом гостя и подходом следующего человека.

---

## Что делает проект

Скрипт:

1. открывает видео;
2. даёт вручную выбрать один столик на первом кадре;
3. детектирует людей на каждом кадре;
4. проверяет, находится ли человек в зоне выбранного столика;
5. фиксирует события:
   - `table_empty`
   - `table_occupied`
6. сохраняет события в `CSV`;
7. считает среднее время между уходом гостя и подходом следующего человека;
8. сохраняет/показывает видео с визуализацией состояния столика.

## Запуск

Положите видео в папку videos/ и запустите:

`python main.py --video video1.mp4`

Важно: в скрипте ожидается, что видео лежит по пути:
`videos/video1.mp4`

Как работает логика детекции
### 1. Выбор столика

На первом кадре пользователь вручную отмечает границы столика мышкой.
После этого координаты сохраняются как полигон зоны анализа.

### 2. Детекция людей

На каждом кадре используется `YOLOv8n` для поиска людей.
Для каждого обнаруженного человека берётся `bounding box` и центр объекта.

### 3. Проверка пересечения с зоной столика

Если центр человека или область вокруг него попадает в полигон столика, считается, что стол занят.

### 4. Фиксация событий

Скрипт отслеживает два основных перехода состояния:
```
стол был пустой → появился человек → table_occupied
стол был занят → человек ушёл → table_empty
```
Для устойчивости используется простая защита от дребезга через временной порог.

В этот раздел можно добавить скриншот сложного кадра, где:

- человек частично закрыт другим объектом;
- человек одновременно определяется несколько раз;
- несколько людей попадают в кадр одновременно.

## Ограничения прототипа
- анализируется только один столик;
- не различаются гости и сотрудники;
- не используется полноценное трекинг-решение;
- качество результата зависит от ракурса камеры и видимости зоны столика;
- возможны ошибки при частичных перекрытиях и быстрых движениях.

## Итог

Проект демонстрирует базовый рабочий сценарий анализа видео для контроля состояния столика без обучения тяжёлых моделей.
Решение подходит как прототип, на основе которого можно дальше развивать полноценную систему аналитики.
